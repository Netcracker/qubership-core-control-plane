# Cloud Core Mesh traffic-policy CRs → Istio DestinationRule

## Overview

This skill converts **standalone** traffic-policy CRs to Istio `DestinationRule` resources:

| Source CR | apiVersion | Output |
|---|---|---|
| `StatefulSession` | `nc.core.mesh/v3` | `DestinationRule` with `consistentHash.httpCookie` |
| `LoadBalance` | `nc.core.mesh/v2` or `nc.core.mesh/v3` | `DestinationRule` with `consistentHash.*` |

> **Out of scope:** `StatefulSession` embedded inside a `RouteConfiguration` rule
> is handled by the [`core-mesh-crs-to-gatewayapi`](../core-mesh-crs-to-gatewayapi/SKILL.md) skill
> as part of HTTPRoute generation.

The transformation keeps charts deployable to **both mesh types** by wrapping originals in
`{{- if eq .Values.SERVICE_MESH_TYPE "Core" }}` and generated resources in
`{{- if eq .Values.SERVICE_MESH_TYPE "Istio" }}`.

---

## Scope

Only files containing a standalone `StatefulSession` or `LoadBalance` CR document.
Do not touch Deployments, Services, helpers (`*.tpl`), or any non-traffic-policy file.

---

## Step-by-Step Procedure

### Step 1 — Discover

```bash
grep -rl \
  -e 'kind: StatefulSession' \
  -e 'kind: LoadBalance' \
  --include="*.yaml" --include="*.yml" \
  <folder>
```

List each discovered file and its contained kinds before proceeding.

**Scope gate:** confirm `apiVersion` is `nc.core.mesh/v2` or `nc.core.mesh/v3` before treating
a file as in scope. Drop false positives.

### Step 2 — Wrap originals in Core condition

Wrap each CR document in the original file:

```yaml
{{- if eq .Values.SERVICE_MESH_TYPE "Core" }}
apiVersion: nc.core.mesh/v3
kind: StatefulSession
# ... original content unchanged ...
{{- end }}
```

For multi-document YAML files: wrap each document individually.

### Step 3 — Generate Istio files

Create a new `-istio` sibling file for each original:

```
templates/stateful-session.yaml  → templates/stateful-session-istio.yaml
templates/load-balance.yaml      → templates/load-balance-istio.yaml
```

Each autogenerated file must be wrapped in the ISTIO guard:

```yaml
{{- if eq .Values.SERVICE_MESH_TYPE "Istio" }}
# ... DestinationRule resources ...
{{- end }}
```

**StatefulSession → DestinationRule:** follow [stateful-session-mapping.md](stateful-session-mapping.md).  
**LoadBalance → DestinationRule:** follow [load-balance-mapping.md](load-balance-mapping.md).

### Step 4 — Validation checklist

- [ ] Every original file's CRs are wrapped in `Core` condition
- [ ] Every generated file's resources are wrapped in `Istio` condition
- [ ] `StatefulSession` with cookie → `DestinationRule` generated; delete/disabled requests skipped
- [ ] `StatefulSession` with `hostname`/`port` → `# ⚠ MANUAL REVIEW` comment added (endpoint-level targeting)
- [ ] `LoadBalance` → `DestinationRule` generated from first policy; multi-policy cases get `# ⚠ MANUAL REVIEW` comment
- [ ] `overridden: true` on any CR → `# ⚠ MANUAL REVIEW` comment added
- [ ] No hardcoded values where Helm expressions existed
- [ ] YAML is valid

### Fields that MUST be flagged with `# ⚠ MANUAL REVIEW`

| Source | Field | Trigger |
|---|---|---|
| `StatefulSession.spec` | `hostname` | non-empty |
| `StatefulSession.spec` | `port` | non-empty |
| `StatefulSession.spec` | `overridden` | non-empty |
| `LoadBalance.spec` | `overridden` | non-empty |
| `LoadBalance.spec.policies` | more than one entry | — |
